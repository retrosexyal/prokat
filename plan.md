# План разработки: сервис аренды товаров (Next.js + MongoDB)

## 0) Цели и MVP
**Цель:** сайт для аренды товаров с онлайн бронированием по датам.

### MVP (первый релиз)
- Каталог товаров (список + карточка товара)
- Календарь занятости товара (недоступные даты)
- Создание бронирования (пользователь выбирает даты + контактные данные)
- Проверка доступности (нельзя забронировать пересекающиеся даты)
- Статусы брони: `pending` → `confirmed` / `cancelled`
- Админка (минимальная): добавление/редактирование товаров
- Хранение в MongoDB (на старте допускается мок для товаров)
- Базовая авторизация для админа

### Не-MVP (после)
- Оплата онлайн (Stripe/ЮKassa)
- Доставка/самовывоз, залог
- Система сообщений/уведомлений (email/telegram)
- Отзывы, рейтинги
- Мульти-склад/локации

## 2) Модель данных (MongoDB)
### Item (товар)
- `_id`
- `title` (string)
- `description` (string)
- `pricePerDay` (number)
- `deposit` (number, optional)
- `images` (string[])
- `category` (string)
- `quantity` (number) // если несколько одинаковых единиц
- `active` (boolean)
- `createdAt`, `updatedAt`

### Booking (бронь)
- `_id`
- `itemId` (ObjectId)
- `startDate` (Date)
- `endDate` (Date) // включительно или исключая последний день — решить сейчас
- `status` (`pending` | `confirmed` | `cancelled`)
- `customer`:
  - `name` (string)
  - `phone` (string)
  - `email` (string, optional)
- `totalPrice` (number)
- `createdAt`, `updatedAt`

### Правило пересечения дат
Пересечение есть, если:
- `newStart <= existingEnd` И `newEnd >= existingStart`
(при условии, что endDate включительно)
