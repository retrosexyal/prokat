## Подключить аналитику и события бронирования/звонка.

В src/app/api/bookings/route.ts в POST нет проверки ObjectId.isValid(productId) перед new ObjectId(productId). Некорректный productId может давать 500 вместо нормального 400.

Там же нет серверной проверки, что дата начала не в прошлом. В интерфейсе это закрыто, но API тоже должен защищаться.

В src/app/api/images/route.ts любой авторизованный пользователь может отправить upload, а DELETE удаляет по publicId без проверки владельца. Это риск для Cloudinary и чужих изображений.

В src/app/api/products/route.ts серверно стоит жёстче валидировать цены, minDays, deposit: сейчас часть значений можно попытаться отправить напрямую в обход UI.
